# Mantine UI Standards

## [MANT-1] Required Packages

- **Required**: Yes
- **Summary**: Include required Mantine packages.
- **Details**:
  - `@mantine/core` for UI components
  - `@mantine/hooks` for utility hooks
  - `tailwind-preset-mantine` for Tailwind CSS integration
  - Configure Next.js to optimize Mantine imports

## [MANT-2] TypeScript Integration

- **Required**: Yes
- **Summary**: Use proper TypeScript patterns with Mantine.
- **Details**:
  - Import component props types by adding `Props` to the component name
  - For regular components, extend the component's Props interface
  - For polymorphic components, use `ElementProps` utility type
  - Use Mantine utility types like `MantineSize` and `MantineColorScheme`

## [MANT-3] Theme Configuration

- **Required**: Yes
- **Summary**: Create a standardized theme configuration.
- **Details**:
  - Create a shared theme in a dedicated `theme.ts` file
  - Use `MantineThemeOverride` type for type safety
  - Define custom colors and breakpoints consistently
  - Include `ColorSchemeScript` in the document head
  - Wrap your application with `MantineProvider`

## [MANT-4] Theme Type Extensions

- **Required**: No
- **Summary**: Extend theme types properly when needed.
- **Details**:
  - Create `mantine.d.ts` file to extend theme types
  - Extend `MantineTheme` interface for custom properties
  - Extend `MantineThemeOther` for additional theme properties
  - Extend `MantineThemeColorsOverride` for custom colors

## [MANT-5] Tailwind Integration

- **Required**: Yes
- **Summary**: Integrate Mantine with Tailwind CSS.
- **Details**:
  - Use `tailwind-preset-mantine` for integration
  - Include the autogenerated `theme.css` file in your application
  - Follow the mapping pattern from Mantine CSS variables to Tailwind variables
  - Use Tailwind classes that map to Mantine variables

## [MANT-6] Component Usage

- **Required**: Yes
- **Summary**: Use Mantine components correctly.
- **Details**:
  - Use direct imports for better tree-shaking
  - Import component props types for strong typing
  - Extend components properly using TypeScript
  - For regular components, extend the component's Props interface
  - For polymorphic components, use `ElementProps` utility type

## [MANT-7] Server vs. Client Components

- **Required**: Yes
- **Summary**: Handle server and client components correctly.
- **Details**:
  - Mark components that use Mantine hooks as client components with "use
    client"
  - Server components can still render Mantine components without hooks
  - Use proper data passing between server and client components
  - Consider the performance implications of client vs. server components

## [MANT-8] Accessibility

- **Required**: Yes
- **Summary**: Ensure accessibility of Mantine components.
- **Details**:
  - Ensure color combinations meet WCAG AA standards
  - Maintain focus styles on interactive elements
  - Ensure all interactive components are keyboard accessible
  - Include appropriate ARIA attributes as needed

## [MANT-9] Testing

- **Required**: Yes
- **Summary**: Test Mantine components properly.
- **Details**:
  - Test Mantine components with provided testing utilities
  - Set up proper theme providers in tests
  - Test both light and dark modes when relevant
  - Use `@repo/testing/vitest/mantine` for test utilities
