# 🔐 Authentication System

Forge uses **Better Auth** to provide a comprehensive authentication system with organizations, API
keys, social providers, and advanced security features.

import { Callout } from 'nextra/components';

<Callout type="info">
  The auth system is built on `@repo/auth` package and integrated with analytics for complete
  tracking and feature flag support.
</Callout>

## Core Features

- **Multiple Sign-in Methods**: Email/password, magic links, social providers (Google, GitHub,
  Discord)
- **Organizations & Teams**: Multi-tenant architecture with role-based permissions
- **API Key Management**: Generate and manage API keys with rate limiting
- **Two-Factor Authentication**: Optional or required 2FA with TOTP
- **Passkey Support**: WebAuthn for passwordless authentication
- **Magic Links**: Secure email-based authentication
- **Impersonation**: Admin ability to impersonate users
- **Session Management**: Secure session handling with middleware

## Configuration

### Server Configuration

```typescript
// packages/auth/server.ts
import { betterAuth } from 'better-auth';
import { magicLink, organization, twoFactor } from 'better-auth/plugins';

export const auth = betterAuth({
  database: {
    provider: 'postgresql',
    url: process.env.DATABASE_URL,
  },
  emailAndPassword: {
    enabled: true,
    autoSignIn: true,
  },
  plugins: [
    magicLink({
      sendMagicLink: async ({ url, token, user }) => {
        await sendMagicLinkEmail({ url, token, user });
      },
    }),
    organization({
      allowUserToCreateOrganization: true,
      organizationLimit: 5,
    }),
    twoFactor({
      totp: {
        period: 30,
        window: 1,
      },
    }),
  ],
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID,
      clientSecret: process.env.GITHUB_CLIENT_SECRET,
    },
    google: {
      clientId: process.env.GOOGLE_CLIENT_ID,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET,
    },
  },
});
```

## Route Protection

### Middleware Protection

```typescript
// middleware.ts
import { createAuthMiddleware } from '@repo/auth/server-utils';
import {
  noseconeMiddleware,
  noseconeOptions,
  noseconeOptionsWithToolbar,
} from '@repo/security/middleware';

const securityHeaders = env.FLAGS_SECRET
  ? noseconeMiddleware(noseconeOptionsWithToolbar)
  : noseconeMiddleware(noseconeOptions);

// Create auth middleware with API key support
const authMiddleware = createAuthMiddleware({
  apiKeyHeaders: ['x-api-key'],
  publicApiRoutes: ['/api/auth'], // Only auth endpoints are public
  publicWebRoutes: [
    '/sign-in',
    '/sign-up',
    '/forgot-password',
    '/reset-password',
    '/_next',
    '/favicon.ico',
    '/.well-known',
  ],
  redirectPath: '/sign-in',
});

export default authMiddleware(() => {
  securityHeaders();
  return NextResponse.next();
}) as unknown as NextMiddleware;

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
  runtime: 'nodejs',
};
```

### Component Protection

```typescript

  return <>{children}</>;
}
```

## API Keys

### Two Types of API Keys

Forge supports two types of API keys for different use cases:

1. **User API Keys** - Generated through Better Auth for individual users
2. **Service API Keys** - Hardcoded keys for service-to-service authentication

### User API Keys

```typescript
import { generateApiKey } from '@repo/auth/server';

const apiKey = await generateApiKey({
  userId: 'user-123',
  name: 'Production API Key',
  scopes: ['read:users', 'write:posts'],
  expiresAt: new Date(Date.now() + 365 * 24 * 60 * 60 * 1000), // 1 year
});
```

### Service API Keys

Service API keys are configured via environment variables and can be rotated through Doppler:

```bash
# .env.local
SERVICE_API_KEY=your-32-character-minimum-secure-key

# Generate a secure key:
openssl rand -base64 32
```

### API Key Authentication

All API routes in Forge apps require authentication. The unified auth helper checks both service and
user API keys:

```typescript
// app/lib/auth.ts
import { NextResponse } from 'next/server';
import { auth } from '@repo/auth/server';
import { env } from '../../env';

export async function requireAuth(request: Request) {
  // Check for service API key first (for service-to-service auth)
  const providedApiKey = request.headers.get('x-api-key');

  if (providedApiKey && env.SERVICE_API_KEY) {
    if (providedApiKey === env.SERVICE_API_KEY) {
      return {
        user: {
          id: 'service',
          email: 'service@system',
          name: 'Service Account',
        },
        session: {
          id: 'service-session',
          userId: 'service',
          activeOrganizationId: 'system',
        },
      };
    }
  }

  // Otherwise, check for user session or user API key
  const session = await auth.api.getSession({
    headers: request.headers,
  });

  if (!session) {
    return NextResponse.json(
      {
        error: 'Unauthorized',
        message: 'Please provide a valid API key via x-api-key header or authenticate via session',
      },
      {
        status: 401,
        headers: {
          'WWW-Authenticate': 'Bearer realm="api", charset="UTF-8"',
        },
      }
    );
  }

  return session;
}
```

### Using API Keys in Routes

```typescript
// app/api/protected/route.ts
import { requireAuth } from '@/lib/auth';

export async function GET(request: Request) {
  const authResult = await requireAuth(request);

  if (authResult instanceof NextResponse) {
    return authResult; // 401 response
  }

  // Access authenticated user/service info
  const { user, session } = authResult;

  // Protected logic here
  return Response.json({
    message: 'Success',
    userId: user.id,
    isServiceAccount: user.id === 'service',
  });
}
```

## Common Patterns

### Protected Page Component

```typescript
// app/dashboard/page.tsx
import { auth } from '@repo/auth/server';
import { redirect } from 'next/navigation';

export default async function DashboardPage() {
  const session = await auth.api.getSession();

  if (!session) {
    redirect('/sign-in');
  }

  return (
    <div>
      <h1>Welcome, {session.user.name}!</h1>
      <p>Organization: {session.user.activeOrganization?.name}</p>
    </div>
  );
}
```

### Auth State Management

```typescript
// hooks/useAuthState.ts
import { useSession } from '@repo/auth/client';
import { analytics } from '@repo/analytics';

export function useAuthState() {
  const { session, isLoading } = useSession();

  const trackAuthEvent = (event: string, data?: object) => {
    analytics.capture(event, {
      userId: session?.user?.id,
      organizationId: session?.user?.activeOrganizationId,
      ...data,
    });
  };

  return {
    user: session?.user,
    organization: session?.user?.activeOrganization,
    isAuthenticated: !!session,
    isLoading,
    trackAuthEvent,
  };
}
```
