# Feature Flags Package

**Note**: Feature flag functionality has been integrated into the `@repo/analytics` package. This
package is deprecated.

## Migration

Feature flags are now part of the analytics package for better integration with PostHog analytics:

```typescript
// Old (deprecated)
import { flag } from '@repo/feature-flags';

// New (current)
import { flag, useFlag } from '@repo/analytics';
```

## Overview

Feature flags were moved to the analytics package to:

- Integrate with PostHog feature flag tracking
- Reduce circular dependencies
- Simplify the package architecture

## Current Usage

See the [Analytics package documentation](/docs/packages/analytics) for current feature flag
implementation.

## Local Development

Use the `LOCAL_FLAGS` environment variable to override feature flags during development:

```bash
LOCAL_FLAGS="new-feature=true,beta-ui=false" pnpm dev
```
