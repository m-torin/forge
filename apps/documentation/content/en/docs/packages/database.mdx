# Database Package

Multi-provider database abstraction layer with Prisma ORM, Redis, Firestore, and vector database
support.

## Overview

The database package provides a comprehensive database infrastructure with multiple provider
support. Features Prisma ORM with PostgreSQL as the primary database, plus adapter patterns for
Redis, Firestore, and Upstash Vector databases for specialized use cases.

## Key Features

- **Prisma ORM with PostgreSQL**: Primary database with comprehensive schema for auth, PIM, and
  workflows
- **Better Auth Integration**: Complete user management, organizations, API keys, and sessions
- **Product Information Management**: Barcode scanning, product catalogs, and asset management
- **Workflow Orchestration**: Workflow configuration, execution tracking, and scheduling
- **Vector Database**: Upstash Vector for AI/ML similarity search and embeddings
- **Redis Cache**: Upstash Redis for caching, rate limiting, and real-time data
- **Firestore Support**: NoSQL document database adapter for flexible data models
- **Type Safety**: Full TypeScript integration across all providers
- **Environment Management**: Doppler integration with local development fallbacks

### Redis (Upstash) - Caching & Real-time

```typescript
import { redis } from '@repo/database/redis';

// Basic key-value operations
await redis.set('user:123', { name: 'John', email: 'john@example.com' });
const user = await redis.get('user:123');

// Lists and queues
await redis.lpush('notifications', 'New message');
const notification = await redis.rpop('notifications');

// Sets for unique collections
await redis.sadd('active_users', 'user:123');
const activeUsers = await redis.smembers('active_users');

// Sorted sets for rankings/leaderboards
await redis.zadd('leaderboard', 100, 'user:123');
const topUsers = await redis.zrange('leaderboard', 0, 9, 'WITHSCORES');

// Hash operations for complex objects
await redis.hset('session:abc', {
  userId: '123',
  expires: Date.now() + 3600000,
  data: JSON.stringify({ preferences: {} }),
});

// Expiration and TTL
await redis.expire('temp:data', 3600); // 1 hour
const ttl = await redis.ttl('temp:data');

// Atomic operations
await redis.incr('page_views');
await redis.decr('inventory:item123');
```

### Vector Database (Upstash) - AI/ML

```typescript
import { upstash } from '@repo/database/upstash';

// Dense vector operations for similarity search
const vector = [0.1, 0.2, 0.3, ...]; // 1536 dimensions for OpenAI embeddings

await upstash.upsert({
  id: 'product-123',
  vector: vector,
  metadata: {
    title: 'iPhone 15 Pro',
    category: 'electronics',
    price: 999.99,
  },
});

// Similarity search
const results = await upstash.query({
  vector: queryVector,
  topK: 10,
  includeMetadata: true,
  filter: {
    category: { $eq: 'electronics' },
    price: { $lte: 1000 },
  },
});

// Text embedding with built-in models
const embedding = await upstash.embed('Search for smartphones under $1000');

// Hybrid vector operations (dense + sparse)
await upstash.upsert({
  id: 'doc-456',
  vector: denseVector,
  sparseVector: {
    indices: [1, 5, 10],
    values: [0.8, 0.6, 0.9],
  },
  metadata: { type: 'document' },
});

// Namespace support for data partitioning
await upstash.upsert({
  id: 'user-embedding',
  vector: userVector,
  namespace: 'user-profiles',
});

// Batch operations
await upstash.upsertMany([
  { id: 'item1', vector: vector1, metadata: meta1 },
  { id: 'item2', vector: vector2, metadata: meta2 },
]);

// Index statistics
const stats = await upstash.describe();
console.log(`Index has ${stats.vectorCount} vectors`);
```

### Firestore - NoSQL Documents

```typescript
import { firestore } from '@repo/database/firestore';

// Document operations
await firestore.create('users', 'user123', {
  name: 'John Doe',
  email: 'john@example.com',
  preferences: {
    theme: 'dark',
    notifications: true,
  },
});

const user = await firestore.findById('users', 'user123');

// Query operations
const activeUsers = await firestore.findMany('users', {
  where: { active: true },
  orderBy: [{ field: 'createdAt', direction: 'desc' }],
  limit: 50,
});

// Transactions
await firestore.transaction(async (tx) => {
  const user = await tx.findById('users', 'user123');
  await tx.update('users', 'user123', {
    loginCount: (user.loginCount || 0) + 1,
    lastLogin: new Date(),
  });
});

// Batch operations
const batch = firestore.batch();
batch.create('posts', 'post1', postData1);
batch.create('posts', 'post2', postData2);
await batch.commit();
```

## Adapter Pattern Usage

Each database provider implements a common adapter interface:

```typescript
// Database adapters for consistent interfaces
import { createPrismaAdapter } from '@repo/database/prisma';
import { createRedisAdapter } from '@repo/database/redis';
import { createFirestoreAdapter } from '@repo/database/firestore';

// Use adapters for provider abstraction
const adapter = createPrismaAdapter();
const items = await adapter.findMany('user', { where: { active: true } });
```

## Error Handling

```typescript
import { Prisma } from '@prisma/client';

try {
  await prisma.user.create({ data: userData });
} catch (error) {
  if (error instanceof Prisma.PrismaClientKnownRequestError) {
    if (error.code === 'P2002') {
      // Unique constraint violation
      throw new Error('User with this email already exists');
    }
  }
  throw error;
}
```
