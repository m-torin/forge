# Analytics Package

Universal analytics system with multi-provider support, integrated feature flags, and development
tools.

## Overview

The analytics package provides a comprehensive solution combining multi-provider analytics tracking
with a complete feature flag management system. It includes development tools, typed flag helpers,
and universal analytics emitters for enterprise-grade observability.

## Key Features

- **Multi-Provider Analytics**: PostHog, Vercel Analytics, Google Analytics, Segment
- **Complete Feature Flag System**: Client/server flags with React hooks and typed helpers
- **Universal Analytics Emitters**: Segment-compatible multi-provider analytics system
- **Vercel Development Toolbar**: Visual feature flag management in development
- **Local Development Overrides**: `LOCAL_FLAGS` environment variable support
- **Type-Safe Flag Helpers**: Domain-specific flag functions (auth, payments, AI, etc.)
- **Privacy-Compliant Tracking**: Configurable consent management
- **Server and Client Implementations**: Full SSR and client-side support

## Feature Flags

### Client-Side Flags

```typescript
import { flag, useFlag, flags } from '@repo/analytics';

// Async flag check
const isEnabled = await flag('ai.enabled', userId);

// React hook for flags
function MyComponent() {
  const aiEnabled = useFlag('ai.enabled');
  const darkMode = useFlag('ui.darkMode');

  return (
    <div>
      {aiEnabled && <AIFeature />}
      <div className={darkMode ? 'dark' : 'light'}>
        Content
      </div>
    </div>
  );
}

// Access all flags
const allFlags = await flags();
```

### Server-Side Flags

```typescript
import { flag, flags } from '@repo/analytics/server';

// API route or server component
export async function GET(request: Request) {
  const userId = getUserId(request);
  const hasOrganizations = await flag('auth.organizations', userId);

  if (hasOrganizations) {
    // Feature-specific logic
  }

const userFlags = await flags(userId);
```

### Typed Flag Helpers

```typescript
import { getAuthFlags, getPaymentFlags, getAIFlags } from '@repo/analytics';

// Domain-specific flag helpers with TypeScript interfaces
```

## Analytics Tracking

### Event Tracking

```typescript
import { track, identify, page } from '@repo/analytics';

// Track custom events
track('button_clicked', {
  button: 'signup',
  page: 'homepage',
  user_plan: 'free',
});

// Identify users
identify(userId, {
  email: 'user@example.com',
  plan: 'pro',
  created_at: new Date(),
});

// Page tracking
page('Dashboard', {
  section: 'analytics',
  feature_flags: await flags(userId),
});
```

### Universal Analytics Emitters

```typescript
import { Analytics } from '@repo/analytics/emitters';

// Configure multiple providers
const analytics = new Analytics({
  providers: {
    segment: {
      writeKey: process.env.SEGMENT_WRITE_KEY,
    },
    posthog: {
      apiKey: process.env.POSTHOG_API_KEY,
      host: process.env.POSTHOG_HOST,
    },
    googleAnalytics: {
      measurementId: process.env.GA_MEASUREMENT_ID,
    },
  },

// Universal tracking API
await analytics.track('purchase_completed', {
  revenue: 99.99,
  currency: 'USD',
  items: ['pro_plan'],
});
```

## Integration Patterns

### Next.js App Router

```typescript
// app/providers.tsx
import { AnalyticsProvider } from '@repo/analytics';

export function Providers({ children }) {
  return (
    <AnalyticsProvider>
      {children}
    </AnalyticsProvider>
  );
}

// app/api/analytics/route.ts
import { flag } from '@repo/analytics/server';

export async function POST(request: Request) {
  const { event, properties } = await request.json();
  const userId = await getUserId(request);

  // Check if analytics is enabled for user
  const analyticsEnabled = await flag('analytics.enabled', userId);

  if (!analyticsEnabled) {
    return new Response('Analytics disabled', { status: 204 });
  }

  await track(event, properties, { userId });
  return new Response('OK');
}
```

### Advanced Flag Usage

```typescript

const rolloutPercentage = await flag('ai.rollout_percentage', userId); // Returns 0-100

const currentHour = new Date().getHours();
const shouldShowSupport = !isBusinessHours || (currentHour >= 9 && currentHour <= 17);

// User segment flags
const userSegment = await flag('user.segment', userId); // Returns 'free', 'pro', 'enterprise'
const features = getFeaturesBySegment(userSegment);
```

## Performance & Optimization

- **Lazy loading**: Providers are loaded only when needed
- **Request batching**: Multiple flag checks are batched for efficiency
- **Caching**: Flag values are cached with configurable TTL
- **Edge compatibility**: Server functions work on Vercel Edge Runtime
- **Bundle optimization**: Client and server code is properly split
