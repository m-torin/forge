# Design System Package

Multi-framework design system supporting three distinct UI approaches: Mantine-based components,
e-commerce library, and search interfaces.

## Overview

The design system package provides a comprehensive collection of UI components organized into three
specialized subsystems. Each system serves different use cases while maintaining design consistency
across web and specialized applications like e-commerce and search.

## Key Features

- **UIX System (Primary)**: Mantine v8-based components with shadcn/ui naming conventions
- **Ciseco E-commerce**: Complete e-commerce UI library with 100+ components
- **Algolia Search**: Specialized search and autocomplete components
- **Authentication Integration**: Built-in auth components with @repo/auth integration
- **Theme System**: Unified theming across all frameworks with dark/light mode
- **TypeScript-First**: Comprehensive type definitions and IntelliSense support
- **Testing Infrastructure**: Vitest + React Testing Library integration

## Installation

```bash
pnpm add @repo/design-system
```

## Component Systems

The main design system built on Mantine v8 with shadcn/ui naming conventions:

```typescript

  Button,
```

**Key Features:**

- Geist font integration for modern typography

## Advanced Usage Patterns

### Responsive Design

```typescript
import { useMediaQuery } from '@repo/design-system/uix';

function ResponsiveComponent() {
  const isMobile = useMediaQuery('(max-width: 768px)');

  return (
    <Card>
      {isMobile ? (
        <MobileLayout />
      ) : (
        <DesktopLayout />
      )}
    </Card>
  );
}
```

### Form Handling

```typescript
import { useForm } from '@repo/design-system/uix';

function ContactForm() {
  const form = useForm({
    initialValues: {
      name: '',
      email: '',
      message: '',
    },
    validate: {
      email: (value) => /^\S+@\S+$/.test(value) ? null : 'Invalid email',
    },
  });

  return (
    <form onSubmit={form.onSubmit(handleSubmit)}>
      <Input
        label="Name"
        {...form.getInputProps('name')}
      />

      <Input
        label="Email"
        type="email"
        {...form.getInputProps('email')}
      />

      <Textarea
        label="Message"
        {...form.getInputProps('message')}
      />

      <Button type="submit">
        Send Message
      </Button>
    </form>
  );
}
```

### Custom Components

```typescript
// Extending base components
import { Button, ButtonProps } from '@repo/design-system/uix';

interface LoadingButtonProps extends ButtonProps {
  loading?: boolean;
}

export function LoadingButton({ loading, children, ...props }: LoadingButtonProps) {
  return (
    <Button
      {...props}
      disabled={loading || props.disabled}
      leftIcon={loading ? <Spinner size="sm" /> : props.leftIcon}
    >
      {loading ? 'Loading...' : children}
    </Button>
  );
}
```

## Migration Guide

### From Direct Mantine Usage

```typescript
// Before
import { Button } from '@mantine/core';

// After
import { Button } from '@repo/design-system/uix';

// API remains the same
<Button variant="filled" size="md">
  Click me
</Button>
```

### Adding Custom Components

```typescript
// Create in appropriate subsystem
// For web: /uix/components/
// For e-commerce: /ciseco/components/
// For search: /algolia/components/

export { MyCustomComponent } from './MyCustomComponent';
```

## Best Practices

1. **Use UIX for general web components** - Primary choice for most web applications
2. **Use Ciseco for e-commerce** - Specialized components for online stores
3. **Use Algolia for search** - Search interfaces and autocomplete
4. **Maintain consistency** - Use the same subsystem throughout an application
5. **Test components** - All custom components should include tests
6. **Document in Storybook** - Add stories for reusable components

The design system provides a comprehensive foundation for building consistent, accessible, and
performant user interfaces across all applications in the Forge platform.
