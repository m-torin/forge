# TypeScript Config

Shared TypeScript configurations for consistent type checking across the monorepo.

## Installation

```bash
pnpm add -D @repo/config
```

## Available Configurations

### Base Configuration

```json
// tsconfig.json
{
  "extends": "@repo/config/typescript/base.json"
}
```

### Next.js Configuration

```json
// tsconfig.json
{
  "extends": "@repo/config/typescript/nextjs.json"
}
```

### React Library Configuration

```json
// tsconfig.json
{
  "extends": "@repo/config/typescript/react-library.json"
}
```

## Configuration Details

### Base Config Features

- **Strict Mode**: Full TypeScript strict checking enabled
- **ESM Support**: Modern ES2022 module resolution
- **Path Mapping**: Support for `@/*` imports
- **Source Maps**: Enabled for debugging

### Next.js Config Features

- Extends base configuration
- Next.js specific compiler options
- JSX support with React 18+
- Incremental compilation

### React Library Config

- Optimized for React component libraries
- Declaration file generation
- Composite project support

## Best Practices

1. **Always extend** rather than override configurations
2. **Use strict mode** for better type safety
3. **Configure paths** in your local tsconfig for module resolution
4. **Enable skipLibCheck** for faster builds in development

## Example Usage

```typescript
// packages/auth/tsconfig.json
{
  "extends": "@repo/config/typescript/base.json",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```
