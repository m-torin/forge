# Authentication Package

Enterprise-grade authentication built on Better Auth v1.2.8 with organizations, API keys, admin
features, and multi-tenant support.

## Overview

The auth package provides comprehensive authentication infrastructure with Better Auth, including
organization management, API key systems, role-based permissions, and admin tools. Designed for
production-scale applications with enterprise requirements.

## Key Features

- **Complete Auth Methods**: Email/password, magic links, social OAuth (Google, GitHub), two-factor
  auth, passkeys
- **Organization System**: Multi-tenant organizations with teams, roles, and member management
- **API Key Management**: Production-ready API authentication with rate limiting and permissions
- **Admin Panel**: User management, impersonation, session control, and ban/unban functionality
- **Role-Based Access Control**: Organization roles (owner, admin, member) and admin roles
  (super-admin, moderator, support)
- **Email Integration**: Welcome emails, password reset, verification, and notifications
- **Analytics Integration**: PostHog tracking for user lifecycle and authentication events
- **React 19 Compatible**: Full compatibility with latest React and Next.js versions

## Session Management

### Client-Side Session

```tsx
import { useSession } from '@repo/auth';

function MyComponent() {
  const { data: session, isPending } = useSession();

  if (isPending) return <Loading />;
  if (!session) return <SignInButton />;

  return (
    <div>
      <p>Welcome, {session.user.name}!</p>
      <p>Organization: {session.activeOrganization?.name}</p>
    </div>
  );
}
```

### Server-Side Session

```typescript
import { auth } from '@repo/auth/server';

// API route
export async function GET(request: Request) {
  const session = await auth.api.getSession({
    headers: request.headers,
  });

  if (!session) {
    return new Response('Unauthorized', { status: 401 });
  }

  return Response.json({ user: session.user });
}

// Server component
async function ServerComponent() {
  const session = await auth.api.getSession();

  if (!session) redirect('/sign-in');

  return <Dashboard user={session.user} />;
}
```

## API Key Management

### Creating API Keys

```typescript
import { apiKey } from '@repo/auth/client';

// Create API key
const key = await apiKey.create({
  name: 'Production API',
  expiresAt: new Date(Date.now() + 365 * 24 * 60 * 60 * 1000), // 1 year
  permissions: ['read:users', 'write:posts'],
  metadata: { environment: 'production' },
});

// Key format: forge_xxxxxxxxxxxxxxxxxxxx
console.log(key.key); // Only shown once!
```

### API Key Authentication

```typescript
// Server-side API validation
import { validateApiKey } from '@repo/auth/server';

export async function GET(request: Request) {
  const apiKey = request.headers.get('x-api-key');

  if (!apiKey) {
    return new Response('API key required', { status: 401 });
  }

  const validation = await validateApiKey(apiKey);

  if (!validation.valid) {
    return new Response('Invalid API key', { status: 401 });
  }

  const { user, organization, permissions } = validation;

  // Rate limiting (100 requests/day by default)
  if (validation.rateLimitExceeded) {
    return new Response('Rate limit exceeded', { status: 429 });
  }

  return Response.json({ data: 'Protected resource' });
}
```

## Permissions & Roles

### Organization Roles

```typescript

// Check permissions
const canManageBilling = hasPermission(userRole, 'manage:billing');
```

## Admin Features

### User Management

```typescript
import { adminActions } from '@repo/auth/server';

// Create user
const user = await adminActions.createUser({
  email: 'admin@example.com',
  password: 'secure-password',
  name: 'Admin User',
  role: 'moderator',
});

// Ban user
await adminActions.banUser({
  userId: 'user-id',
  reason: 'Violated terms of service',
  banUntil: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000), // 7 days
});

// Unban user
await adminActions.unbanUser({ userId: 'user-id' });

// Delete user and data
await adminActions.deleteUser({ userId: 'user-id' });
```

### User Impersonation

```typescript
// Start impersonation (1-hour limit)
const impersonationSession = await adminActions.impersonateUser({
  userId: 'target-user-id',
  adminId: 'admin-user-id',
});

// Stop impersonation
await adminActions.stopImpersonation({
  sessionId: impersonationSession.id,
});
```

### Session Management

```typescript
// List user sessions
const sessions = await adminActions.listUserSessions({
  userId: 'user-id',
});

// Revoke specific session
await adminActions.revokeSession({
  sessionId: 'session-id',
});

// Revoke all user sessions
await adminActions.revokeAllUserSessions({
  userId: 'user-id',
});
```

## Error Handling

```typescript
import { BetterAuthError } from '@repo/auth';

try {
  await signIn.email({ email, password });
} catch (error) {
  if (error instanceof BetterAuthError) {
    switch (error.code) {
      case 'INVALID_CREDENTIALS':
        setError('Invalid email or password');
        break;
      case 'EMAIL_NOT_VERIFIED':
        setError('Please verify your email first');
        break;
      case 'TOO_MANY_REQUESTS':
        setError('Too many attempts. Try again later');
        break;
      default:
        setError('Authentication failed');
    }
  }
}
```
