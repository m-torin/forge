# 🎭 Backstage Admin Panel

Admin panel for managing users, organizations, and system settings with impersonation capabilities.

import { Callout } from 'nextra/components';

<Callout type="info">
  Backstage runs on port **3300** and provides administrative access to manage the entire platform.
</Callout>

## Core Features

- **User Management** - View, edit, and manage user accounts
- **Organization Admin** - Manage organization settings and memberships
- **API Key Oversight** - Monitor and manage API key usage
- **Impersonation** - Safely impersonate users for support
- **Analytics Dashboard** - System metrics and user behavior
- **Feature Flag Management** - Control feature rollouts
- **Billing Management** - Subscription and payment administration
- **System Health** - Monitor service status and performance

## Authentication

Backstage uses the same auth system as other apps with additional admin restrictions:

### Admin Access Requirements

1. User must have `admin` or `owner` role
2. Stricter rate limiting (5 requests/minute for auth)
3. Enhanced security headers and monitoring
4. Audit logging for all actions

### API Authentication

```bash
# Using admin user API key
curl -H "x-api-key: YOUR_ADMIN_API_KEY" \
  https://backstage.app/api/protected/users

# Using service API key (for automation)
curl -H "x-api-key: YOUR_SERVICE_API_KEY" \
  https://backstage.app/api/health
```

## Admin Features

### User Management

```typescript
// Get all users
GET /api/protected/users

  "users": [
    {
      "id": "user-123",

{
  "role": "admin"

// Impersonate user
POST /api/protected/impersonate
{
  "userId": "user-123"
```

## Security

### Protected Routes

All routes require admin authentication:

```typescript
// middleware.ts
const authMiddleware = createAuthMiddleware({
  apiKeyHeaders: ['x-api-key'],
  publicApiRoutes: ['/api/auth'], // Only auth endpoints public
  publicWebRoutes: ['/sign-in', '/sign-up', '/forgot-password', '/reset-password', '/unauthorized'],
  redirectPath: '/sign-in',
});
```

### Rate Limiting

More restrictive limits for admin operations:

| Endpoint Type | Limit       | Window   |
| ------------- | ----------- | -------- |
| Auth          | 5 requests  | 1 minute |
| Admin API     | 30 requests | 1 minute |
| Impersonation | 10 requests | 1 hour   |

### Audit Logging

All admin actions are logged:

```typescript
// Automatic audit logging
analytics.capture('admin_action', {
  action: 'user_role_updated',
  targetUserId: 'user-123',
  adminId: session.user.id,
  changes: { role: 'admin' },
  timestamp: new Date().toISOString(),
});
```

## Stripe Integration

### Webhook Security

Stripe webhooks use signature verification:

```typescript
// Automatically verified
import { stripe } from '@repo/payments/server';

export async function POST(request: Request) {
  const signature = request.headers.get('stripe-signature');
  const event = stripe.webhooks.constructEvent(
    await request.text(),
    signature,
    env.STRIPE_WEBHOOK_SECRET
  );

  // Process webhook event
}
```

### Supported Events

- `checkout.session.completed`
- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.payment_succeeded`
- `invoice.payment_failed`

## Common Admin Tasks

### 1. Managing User Roles

```typescript
// Promote user to admin
await updateUserRole(userId, 'admin');

// Remove admin access
await updateUserRole(userId, 'member');
```

### 2. Monitoring API Usage

```typescript
// Check API key usage
const usage = await getApiKeyUsage(apiKeyId);
console.log(`Requests today: ${usage.requestsToday}`);
console.log(`Rate limit hits: ${usage.rateLimitHits}`);
```

### 3. Impersonating for Support

```typescript
// Start impersonation session
await impersonateUser(userId);

// End impersonation
await endImpersonation();
```
