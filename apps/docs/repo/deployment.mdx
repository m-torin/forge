# Deployment

Forge is designed for modern deployment with Vercel, Docker, or any Node.js
hosting platform with comprehensive secret management.

## Deployment Platforms

### Vercel (Recommended)

Vercel provides the best experience for Next.js applications with automatic
optimizations:

<CodeGroup>
```bash Install & Deploy
# Install Vercel CLI
pnpm add -g vercel

# Deploy an app

cd apps/web vercel

# Deploy with production settings

vercel --prod

````

```bash Quick Deploy
# One-command deploy from app directory
vercel --prod
````

</CodeGroup>

**Vercel Benefits:**

- Zero-config Next.js deployments
- Automatic edge optimization
- Built-in analytics and monitoring
- Seamless Doppler integration

### Docker Deployments

Each app includes Docker support for containerized deployments:

<CodeGroup>
```dockerfile Multi-stage Dockerfile
FROM node:22-alpine AS base
RUN corepack enable

FROM base AS deps WORKDIR /app COPY package.json pnpm-lock.yaml ./ RUN pnpm
install --frozen-lockfile

FROM base AS builder WORKDIR /app COPY --from=deps /app/node_modules
./node_modules COPY . . RUN pnpm build

FROM base AS runner WORKDIR /app ENV NODE_ENV production COPY --from=builder
/app/public ./public COPY --from=builder /app/.next/standalone ./ COPY
--from=builder /app/.next/static ./.next/static

EXPOSE 3000 CMD ["node", "server.js"]

````

```bash Docker Commands
# Build image
docker build -t forge-app .

# Run container
docker run -p 3000:3000 forge-app

# With environment file
docker run --env-file .env.local -p 3000:3000 forge-app
````

</CodeGroup>

### Node.js Hosting

Deploy to any Node.js platform (Railway, Render, Fly.io):

```bash
# Build for production
pnpm build

# Start production server
pnpm start
```

<Note>
  Ensure your hosting platform supports Node.js 22+ and has proper environment
  variable configuration.
</Note>

## Secret Management with Doppler

<Warning>
  **Important**: Forge uses Doppler for centralized secret management across all
  environments. This replaces manual environment variable management.
</Warning>

### Quick Doppler Setup

<AccordionGroup>
  <Accordion title="Install Doppler CLI" icon="download">
    ```bash
    # macOS
    brew install dopplerhq/cli/doppler

    # Linux
    curl -Ls https://cli.doppler.com/install.sh | sh

    # Windows
    scoop install doppler
    ```

  </Accordion>
  
  <Accordion title="Authenticate & Setup" icon="key">
    ```bash
    # Authenticate with Doppler
    doppler login

    # Setup project (run from repo root)

    # Pull all environment variables
    pnpm doppler:pull:all
    ```

  </Accordion>
  
  <Accordion title="Development Workflow" icon="code">
    ```bash
    # Pull latest secrets
    pnpm doppler:pull:all

    pnpm build

    # Run with Doppler (production build)
    pnpm build:doppler
    ```

  </Accordion>
</AccordionGroup>

### CI/CD Integration

#### GitHub Actions with Doppler

```yaml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: dopplerhq/cli-action@v3

      - name: Install dependencies
        run: pnpm install

      - name: Build with Doppler
        env:
          DOPPLER_TOKEN: ${{ secrets.DOPPLER_SERVICE_TOKEN }}
        run: pnpm build:doppler

      - name: Deploy to Vercel
        env:
          VERCEL_TOKEN: ${{ secrets.VERCEL_TOKEN }}
        run: vercel --prod --token=$VERCEL_TOKEN
```

#### Vercel Integration

<Steps>
  <Step title="Remove Manual Variables">
    Remove all environment variables from Vercel dashboard - Doppler will manage
    them
  </Step>
  <Step title="Add Doppler Token">
    Add only `DOPPLER_TOKEN` environment variable to Vercel
  </Step>
  <Step title="Automatic Injection">
    Doppler automatically injects all secrets during build process
  </Step>
</Steps>

## Environment Variables

### Core Required Variables

```bash
# Database
DATABASE_URL="postgresql://user:password@host:port/database"

# Authentication
BETTER_AUTH_SECRET="your-secure-random-secret-minimum-32-chars"

# Service Authentication (for CI/CD and automation)
SERVICE_API_KEY="your-32-character-minimum-secure-api-key"

# Email Service
RESEND_API_KEY="re_your_resend_api_token"
FROM_EMAIL="noreply@yourdomain.com"
```

## Platform-Specific Deployment

### Vercel Deployment

<Steps>
  <Step title="Connect Repository">
    Import your repository in the Vercel dashboard
  </Step>
  <Step title="Configure Build">
    Vercel auto-detects Next.js apps and configures build settings
  </Step>
  <Step title="Add Doppler Token">
    Add `DOPPLER_TOKEN` to environment variables
  </Step>
  <Step title="Deploy">Automatic deployments on every push to main branch</Step>
</Steps>

### Railway Deployment

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login and initialize
railway login
railway init

# Set environment variables
railway variables set DOPPLER_TOKEN=your_token

# Deploy
railway up
```

<Steps>

  </Step>
  <Step title="Environment Variables">

  </Step>
</Steps>

## Deployment Checklist

<Steps>
  <Step title="Environment Setup">
    - [ ] Configure Doppler project - [ ] Set all required environment variables
    - [ ] Test local build with `pnpm build:local`
  </Step>
  <Step title="Security Configuration">
    - [ ] Update `BETTER_AUTH_SECRET` for production - [ ] Configure rate
    limiting with Upstash Redis - [ ] Set up monitoring with Sentry
  </Step>
  <Step title="Service Configuration">
    - [ ] Configure database with proper connection pooling - [ ] Set up email
    service (Resend) - [ ] Configure analytics (PostHog)
  </Step>
  <Step title="Domain & SSL">
    - [ ] Configure custom domain - [ ] Verify SSL certificate - [ ] Update
    `NEXT_PUBLIC_APP_URL`
  </Step>

<Note>
  All Forge applications are designed to be stateless and horizontally scalable.
  Database connections are properly pooled, and no local state is stored in the
  application instances.
</Note>

## Troubleshooting

### Common Deployment Issues

<AccordionGroup>
  <Accordion title="Build Failures" icon="exclamation-triangle">
    **Symptoms**: Build fails during TypeScript compilation or dependency installation
    **Solutions**: ```bash # Check for circular dependencies pnpm madge --circular # Clear cache and
    reinstall pnpm clean && pnpm install # Verify TypeScript compilation pnpm typecheck ```
  </Accordion>

<Accordion title="Environment Variable Issues" icon="key">
  **Symptoms**: Missing environment variables or configuration errors
  **Solutions**: ```bash # Verify Doppler setup doppler setup --no-interactive #
  Check environment validation pnpm build:local # Debug environment loading
  doppler run -- node -e "console.log(process.env)" ```
</Accordion>

  <Accordion title="Database Connection Issues" icon="database">
    **Symptoms**: Cannot connect to database or connection timeouts **Solutions**: - Verify
    `DATABASE_URL` format - Check network connectivity and firewall rules - Ensure database accepts
    connections from deployment IP - Verify connection pool settings
  </Accordion>
</AccordionGroup>
