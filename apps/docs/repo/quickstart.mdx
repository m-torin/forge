# Get Started

Get up and running with Forge in minutes. This guide walks you through setting
up a complete development environment.

## Prerequisites

- **Node.js 22+** - Required for ESM support
- **pnpm 10.6.3+** - Package manager
- **PostgreSQL** - Database (local or hosted)

## Quick Setup

<AccordionGroup>
  <Accordion icon="github" title="Clone the repository">
    ```bash git clone https://github.com/your-org/forge cd forge ```
  </Accordion>
  <Accordion icon="package" title="Install dependencies">
    ```bash pnpm install ```
  </Accordion>
  <Accordion icon="gear" title="Set up environment variables">
    ```bash cp .env.example .env.local ```
  </Accordion>
  <Accordion icon="database" title="Set up database">
    ```bash pnpm migrate ```
  </Accordion>
  <Accordion icon="rocket" title="Start development">
    ```bash pnpm dev ```
  </Accordion>
</AccordionGroup>

## Environment Variables

Forge uses a standardized **SafeEnv pattern** for environment configuration.
Create `.env.local` files in each app with required variables:

```bash
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/forge"

# Authentication
BETTER_AUTH_SECRET="your-secret-key"
BETTER_AUTH_URL="http://localhost:3200"

# Analytics (optional)
NEXT_PUBLIC_POSTHOG_KEY="phc_your_key"

# Email (optional)
RESEND_API_KEY="re_your_key"
```

<Tip>
  All apps and packages use `env.ts` files with graceful fallbacks to prevent
  white screen crashes. See [Environment
  Configuration](/repo/architecture/environment-configuration) for the complete
  pattern.
</Tip>

## Development Commands

```bash
pnpm dev              # Start all apps
pnpm dev --filter=web # Start specific app
pnpm build            # Build all apps
pnpm test             # Run tests
pnpm lint             # Lint and format
```

## Project Structure

- `/apps/webapp` - Main webapp (Port 3200)
- `/apps/ai-chatbot` - AI Chat interface (Port 3100)
- `/apps/email` - Email preview (Port 3500)
- `/apps/storybook` - Component docs (Port 3700)
- `/apps/docs` - Documentation (Port 3800)
- `/packages/*` - Shared packages (includes database with Prisma Studio)

## Deployment

Forge is designed for modern deployment with Vercel, Docker, or any Node.js
hosting platform with comprehensive secret management.

### Vercel (Recommended)

Vercel provides the best experience for Next.js applications:

```bash
# Install Vercel CLI
pnpm add -g vercel

# Deploy from app directory
cd apps/webapp
vercel --prod
```

### Docker Deployments

Each app includes Docker support:

```bash
# Build image
docker build -t forge-app .

# Run container
docker run -p 3000:3000 forge-app
```

### Required Environment Variables

```bash
# Database
DATABASE_URL="postgresql://user:password@host:port/database"

# Authentication
BETTER_AUTH_SECRET="your-secure-random-secret-minimum-32-chars"

# Email Service
RESEND_API_KEY="re_your_resend_api_token"
FROM_EMAIL="noreply@yourdomain.com"
```

### Secret Management with Doppler

<Warning>
  **Important**: Forge uses Doppler for centralized secret management across all
  environments.
</Warning>

```bash
# Install Doppler CLI
brew install dopplerhq/cli/doppler

# Authenticate and setup
doppler login
pnpm doppler:setup
pnpm doppler:pull:all
```

### Build Commands

- `pnpm build` - Production build with Doppler secrets
- `pnpm build:local` - Local build with `.env.local` files
