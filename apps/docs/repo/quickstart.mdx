# Get Started

Get up and running with Forge in minutes. This guide walks you through setting
up a complete development environment.

## Prerequisites

- **Node.js 22+** - Required for ESM support
- **pnpm 10.6.3+** - Package manager
- **PostgreSQL** - Database (local or hosted)

## Quick Setup

<AccordionGroup>
  <Accordion icon="github" title="Clone the repository">
    ```bash git clone https://github.com/your-org/forge cd forge ```
  </Accordion>
  <Accordion icon="package" title="Install dependencies">
    ```bash pnpm install ```
  </Accordion>
  <Accordion icon="gear" title="Set up environment variables">
    ```bash cp .env.example .env.local ```
  </Accordion>
  <Accordion icon="database" title="Set up database">
    ```bash pnpm migrate ```
  </Accordion>
  <Accordion icon="rocket" title="Start development">
    ```bash pnpm dev ```
  </Accordion>
</AccordionGroup>

## Environment Variables

Forge uses a standardized **SafeEnv pattern** for environment configuration.
Create `.env.local` files in each app with required variables:

```bash
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/forge"

# Authentication
BETTER_AUTH_SECRET="your-secret-key"
BETTER_AUTH_URL="http://localhost:3200"

# Analytics (optional)
NEXT_PUBLIC_POSTHOG_KEY="phc_your_key"

# Email (optional)
RESEND_API_KEY="re_your_key"
```

<Tip>
  All apps and packages use `env.ts` files with graceful fallbacks to prevent
  white screen crashes. See [Environment
  Configuration](/repo/architecture/environment-configuration) for the complete
  pattern.
</Tip>

## Development Commands

```bash
pnpm dev              # Start all apps
pnpm dev --filter=web # Start specific app
pnpm build            # Build all apps
pnpm test             # Run tests
pnpm lint             # Lint and format
```

## Project Structure

- `/apps/docs` - Documentation (Port 3800)
- `/apps/email` - Email preview (Port 3500)
- `/apps/storybook` - Component docs (Port 3700)
- `/apps/studio` - Prisma Studio (Port 3600)
- `/packages/*` - Shared packages
