### Available Configurations

| Configuration   | Use Case                  | Key Features                                                   |
| --------------- | ------------------------- | -------------------------------------------------------------- |
| `next`          | Next.js applications      | React hooks, Next.js specific rules, performance optimizations |
| `react-package` | React component libraries | JSX rules, component patterns, accessibility                   |
| `package`       | Node.js packages          | Import rules, TypeScript integration, performance              |
| `server`        | Server applications       | Node.js best practices, security rules                         |
| `remix`         | Remix applications        | Remix-specific patterns, React Router                          |
| `.` (base)      | Base TypeScript projects  | Core TypeScript, security, imports, code quality               |

### Rule Categories

#### Core Rules

- **TypeScript**: Strict type checking, naming conventions, performance
- **Import/Export**: Module resolution, dependency management, tree-shaking
- **Code Quality**: Complexity, maintainability, readability
- **Security**: XSS prevention, safe patterns, vulnerability detection

#### Framework-Specific Rules

- **React**: Hooks validation, JSX best practices, component patterns
- **Next.js**: App Router patterns, performance optimizations, Server Components
- **Node.js**: File system security, process management, async patterns

#### Testing & Quality Assurance

- **Vitest**: Test structure, assertion preferences, async testing
- **Playwright**: E2E testing patterns, web-first assertions
- **Testing Library**: Query best practices, semantic matchers

#### Code Organization

- **Import Sorting**: Automatic import organization with perfectionist
- **Promise Handling**: Async/await patterns, error handling
- **Accessibility**: WCAG compliance, screen reader support

### Next.js Configuration

Advanced Next.js configuration with performance optimizations, Prisma plugin,
and PostHog analytics:

```typescript
// packages/config/src/next/index.ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  // ESLint configuration
  eslint: {
    ignoreDuringBuilds: true
  },

  // Experimental features
  experimental: {
    typedRoutes: true,
    forceSwcTransforms: true,
    optimizePackageImports: ["@mantine/core", "@mantine/hooks"]
  },

  // Turbopack configuration
  turbopack: {
    rules: {
      "*.svg": {
        as: "*.js",
        loaders: ["@svgr/webpack"]
      }
    }
  },

  // SWC compiler optimizations
  compiler: {
    reactRemoveProperties: isProd,
    removeConsole: isProd ? { exclude: ["error", "warn"] } : false
  },

  // Image optimization
  images: {
    formats: ["image/avif", "image/webp"]
  },

  // PostHog analytics rewrites
  async rewrites() {
    return {
      beforeFiles: [
        {
          source: "/ingest/static/:path*",
          destination: "https://us-assets.i.posthog.com/static/:path*"
        },
        {
          source: "/ingest/:path*",
          destination: "https://us.i.posthog.com/:path*"
        }
      ]
    };
  },

  // Comprehensive webpack configuration with Prisma plugin
  webpack: webpackConfig
};

export default nextConfig;
```

## Migration Guide

### From Custom Configurations

1. **Remove existing configs**:

   ```bash
   rm .eslintrc.js tsconfig.json prettier.config.js
   ```

2. **Install Forge configs**:

   ```bash
   pnpm add -D @repo/eslint-config @repo/config
   ```

3. **Create new config files** using the templates above

4. **Update package.json scripts**:
   ```json
   {
     "scripts": {
       "lint": "eslint . --fix",
       "typecheck": "tsc --noEmit",
       "format": "prettier --write ."
     }
   }
   ```

### Customization

For project-specific needs, extend the base configurations:

```typescript
// eslint.config.ts
import baseConfig from "@repo/eslint-config/next";

export default [
  ...baseConfig,
  {
    rules: {
      // Project-specific overrides
      "no-console": "warn"
    }
  }
];
```

These configuration packages provide a solid foundation for maintaining high
code quality and consistency across the entire Forge monorepo while allowing for
project-specific customizations when needed.
