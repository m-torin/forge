# TypeScript Config Package

Shared TypeScript configurations providing consistent, optimized settings across
all packages and applications in the Forge monorepo.

## Overview

The TypeScript config package provides pre-configured TypeScript settings
optimized for different project types, ensuring consistent type checking,
compilation, and development experience across the entire codebase.

## Installation

```bash
pnpm add -D @repo/config
```

## Available Configurations

<Tabs>
  <Tab title="Next.js Apps">
    For Next.js applications (apps/docs, apps/storybook, etc.):

    ```json
    // tsconfig.json
    {
      "extends": "@repo/config/typescript/nextjs.json",
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "#/*": ["./src/*"]
        }
      },
      "include": [
        "next-env.d.ts",
        "**/*.ts",
        "**/*.tsx",
        ".next/types/**/*.ts"
      ],
      "exclude": ["node_modules"]
    }
    ```

  </Tab>

  <Tab title="React Libraries">
    For React component libraries and UI packages:

    ```json
    // tsconfig.json
    {
      "extends": "@repo/config/typescript/react-library.json",
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "#/*": ["./src/*"]
        }
      },
      "include": ["src/**/*"],
      "exclude": ["node_modules", "dist", "**/*.test.*", "**/*.spec.*"]
    }
    ```

  </Tab>

  <Tab title="Cloudflare Workers">
    For Cloudflare Workers and edge runtime applications:

    ```json
    // tsconfig.json
    {
      "extends": "@repo/config/typescript/cloudflare.json",
      "compilerOptions": {
        "baseUrl": ".",
        "paths": {
          "#/*": ["./src/*"]
        }
      },
      "include": ["src/**/*"],
      "exclude": ["node_modules", "dist"]
    }
    ```

  </Tab>

  <Tab title="Base Config">
    Core configuration for all TypeScript projects:

    ```json
    // base.json
    {
      "compilerOptions": {
        "target": "ES2022",
        "lib": ["ES2022"],
        "allowJs": true,
        "skipLibCheck": true,
        "strict": true,
        "noEmit": true,
        "esModuleInterop": true,
        "module": "ESNext",
        "moduleResolution": "bundler",
        "resolveJsonModule": true,
        "isolatedModules": true,
        "jsx": "preserve",
        "incremental": true,
        "plugins": [
          {
            "name": "next"
          }
        ]
      }
    }
    ```

  </Tab>
</Tabs>

## Best Practices

<Note>
  **Configuration Guidelines:** 1. Always extend from the appropriate base
  configuration 2. Minimize custom overrides - prefer consistent defaults 3. Use
  strict type checking in all environments 4. Enable incremental compilation for
  better performance 5. Keep include/exclude patterns specific and minimal
</Note>

The TypeScript config package ensures consistent, modern TypeScript
configuration across the entire Forge platform, providing excellent developer
experience while maintaining strict type safety and optimal performance.
