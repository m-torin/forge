# Authentication Package

Enterprise-grade authentication built on Better Auth v1.2.12 with organizations,
API keys, admin features, and multi-tenant support.

## Overview

The auth package provides comprehensive authentication infrastructure with
Better Auth, including organization management, API key systems, role-based
permissions, and admin tools. Designed for production-scale applications with
enterprise requirements.

## Authentication Methods

### Email/Password Authentication

```typescript
import { signIn, signUp } from "@repo/auth/client";

// Sign up
const result = await signUp.email({
  email: "user@example.com",
  password: "secure-password",
  name: "John Doe"
});

// Sign in
const session = await signIn.email({
  email: "user@example.com",
  password: "secure-password"
});
```

### Magic Links

```typescript
import { sendMagicLink } from "@repo/auth/client";

// Send magic link (20-minute expiration)
await sendMagicLink({
  email: "user@example.com",
  callbackURL: "/dashboard"
});
```

### Social Authentication

```typescript
import { signIn } from "@repo/auth/client";

// Google OAuth
await signIn.social({
  provider: "google",
  callbackURL: "/dashboard"
});

// GitHub OAuth
await signIn.social({
  provider: "github",
  callbackURL: "/dashboard"
});
```

## Session Management

### Client-Side Session

```tsx
import { useSession } from "@repo/auth";

function MyComponent() {
  const { data: session, isPending } = useSession();

  if (isPending) return <Loading />;
  if (!session) return <SignInButton />;

  return (
    <div>
      <p>Welcome, {session.user.name}!</p>
      <p>Organization: {session.activeOrganization?.name}</p>
    </div>
  );
}
```

### Server-Side Session

```typescript
import { auth } from '@repo/auth/server';

// API route
export async function GET(request: Request) {
  const session = await auth.api.getSession({
    headers: request.headers,
  });

  if (!session) {
    return new Response('Unauthorized', { status: 401 });
  }

  return Response.json({ user: session.user });
}

// Server component
async function ServerComponent() {
  const session = await auth.api.getSession();

  if (!session) redirect('/sign-in');

  return <Dashboard user={session.user} />;
}
```

## Organization Management

### Creating Organizations

```typescript
import { organization } from "@repo/auth/client";

// Create organization (limit: 5 per user)
const newOrg = await organization.create({
  name: "My Company",
  slug: "my-company"
});

// Switch active organization
await organization.setActive({
  organizationId: newOrg.id
});
```

### Managing Members

```typescript
// Invite member (limit: 100 invitations per org)
await organization.inviteMember({
  email: "member@example.com",
  role: "admin", // owner, admin, member
  organizationId: "org-id"
});

// Accept invitation
await organization.acceptInvitation({
  invitationId: "invitation-id"
});

// Update member role
await organization.updateMemberRole({
  userId: "user-id",
  role: "member",
  organizationId: "org-id"
});
```

## API Key Management

```typescript
import { apiKey } from "@repo/auth/client";

// Create API key
const key = await apiKey.create({
  name: "Production API",
  expiresAt: new Date(Date.now() + 365 * 24 * 60 * 60 * 1000), // 1 year
  permissions: ["read:users", "write:posts"],
  metadata: { environment: "production" }
});

// API key authentication
const response = await fetch("/api/data", {
  headers: {
    Authorization: `Bearer ${apiKey}`
  }
});
```

## Security Features

- **Rate Limiting**: Built-in protection against brute force attacks
- **Session Security**: Automatic session rotation and secure cookies
- **CSRF Protection**: Built-in CSRF token validation
- **Email Verification**: Required for account activation
- **Password Requirements**: Configurable strength requirements
- **Account Lockout**: Automatic lockout after failed attempts

The authentication package provides enterprise-grade security and user
management features built on Better Auth, designed for scalable multi-tenant
applications.
