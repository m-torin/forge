# Design System Package

A comprehensive design system providing consistent UI components, design tokens, and patterns across
all applications in the Forge platform.

## Overview

The design system package serves as the foundation for all user interfaces, providing pre-built
components, design tokens, and styling utilities that ensure consistency and accessibility across
the platform.

## Key Features

- **Mantine UI v8 Integration**: Primary UI framework with comprehensive component library
- **Custom Component Library**: Specialized components for Forge-specific use cases
- **Design Token System**: Consistent colors, typography, spacing, and responsive breakpoints
- **Dark Mode Support**: Built-in dark mode with smooth transitions
- **TypeScript First**: Full TypeScript support with comprehensive type definitions
- **Accessibility**: WCAG 2.1 AA compliance with keyboard navigation and ARIA support
- **Responsive Design**: Mobile-first approach with consistent breakpoints
- **Theme Customization**: Flexible theming system for brand customization

    ```typescript
    import { Card, Image, Text, Badge, Group, Button } from '@mantine/core';
    import { forwardRef } from 'react';
    
    export interface ProductCardProps {
      id: string;
      title: string;
      price: number;

            {image && (
              <Card.Section>
                <Image src={image} alt={title} height={200} />
              </Card.Section>
            )}
            
            <Group justify="space-between" mt="md">
              <Text fw={500}>{title}</Text>

              </Badge>
            </Group>

              ${price.toFixed(2)}
            </Text>
            
            <Button 
              variant="light" 
              fullWidth 
              mt="md"
              onClick={() => onEdit?.(id)}
            >
              Edit Product
            </Button>
          </Card>
        );
      }
    );
    ```

### Testing Framework

<CodeGroup>
```typescript Test Setup
// test-utils.tsx - Enhanced testing utilities
import { render, RenderOptions } from '@testing-library/react';
import { MantineProvider } from '@mantine/core';
import { ModalsProvider } from '@mantine/modals';
import { Notifications } from '@mantine/notifications';

interface WrapperProps { children: React.ReactNode; }

function AllTheProviders({ children }: WrapperProps) { return ( <MantineProvider> <ModalsProvider>

<Notifications /> {children} </ModalsProvider> </MantineProvider> ); }

const customRender = ( ui: React.ReactElement, options?: Omit<RenderOptions, 'wrapper'> ) =>
render(ui, { wrapper: AllTheProviders, ...options });

export * from '@testing-library/react';
export { customRender as render };
```

```typescript Component Testing Pattern
// ProductCard.test.tsx - Comprehensive component testing
import { render, screen, fireEvent, waitFor } from './test-utils';
import { ProductCard } from './ProductCard';

describe('ProductCard', () => {
  const mockProduct = {
    id: '1',
    title: 'Test Product',
    price: 29.99,
    status: 'active' as const,
  };

  describe('Rendering', () => {
    it('renders product information correctly', () => {
      render(<ProductCard {...mockProduct} data-testid="product-card" />);

      expect(screen.getByTestId('product-card')).toBeInTheDocument();
      expect(screen.getByText('Test Product')).toBeInTheDocument();
      expect(screen.getByText('$29.99')).toBeInTheDocument();
    });

    it('displays product image when provided', () => {
      const productWithImage = {
        ...mockProduct,
        image: '/test-image.jpg',
      };

      render(<ProductCard {...productWithImage} />);

      const image = screen.getByRole('img', { name: 'Test Product' });
      expect(image).toHaveAttribute('src', '/test-image.jpg');
    });
  });

  describe('Interactions', () => {
    it('calls onEdit when edit button is clicked', async () => {
      const mockOnEdit = jest.fn();

      render(<ProductCard {...mockProduct} onEdit={mockOnEdit} />);

      const editButton = screen.getByRole('button', { name: /edit product/i });
      fireEvent.click(editButton);

      expect(mockOnEdit).toHaveBeenCalledWith('1');
    });

    it('handles keyboard navigation correctly', () => {
      render(<ProductCard {...mockProduct} onEdit={jest.fn()} />);

      const editButton = screen.getByRole('button', { name: /edit product/i });
      editButton.focus();

      expect(document.activeElement).toBe(editButton);
    });
  });

  describe('Accessibility', () => {
    it('has proper ARIA attributes', () => {
      render(<ProductCard {...mockProduct} />);

      const card = screen.getByRole('article');
      expect(card).toBeInTheDocument();

      const statusBadge = screen.getByText('active');
      expect(statusBadge).toHaveAttribute('aria-label', 'Product status: active');
    });

    it('maintains focus management', () => {
      render(<ProductCard {...mockProduct} onEdit={jest.fn()} />);

      const editButton = screen.getByRole('button');
      expect(editButton).toHaveAttribute('type', 'button');
      expect(editButton).not.toHaveAttribute('disabled');
    });
  });

  describe('Responsive Design', () => {
    it('adapts to different screen sizes', () => {
      // Test responsive behavior
      Object.defineProperty(window, 'innerWidth', {
        writable: true,
        configurable: true,
        value: 768,
      });

      render(<ProductCard {...mockProduct} />);

      // Verify mobile-specific layout adjustments
      const card = screen.getByTestId('product-card');
      expect(card).toHaveClass('product-card-responsive');
    });
  });
});
```

## Best Practices

<Warning>
  **Design System Guidelines:** - Always use design tokens instead of hardcoded values - Follow
  Mantine UI patterns for consistency - Include proper TypeScript types for all props - Add
  comprehensive test coverage for custom components - Ensure accessibility compliance for all
  components - Use standardized data-testid attributes for reliable testing
</Warning>

### Recommended Patterns

1. **Component Composition**
   - Use compound components for complex UI patterns
   - Leverage Mantine's built-in components as building blocks
   - Create reusable hooks for common functionality
   - Follow the container/presentational component pattern

2. **Performance Optimization**
   - Use React.memo for expensive components
   - Implement proper key props for lists
   - Lazy load heavy components
   - Optimize bundle size with tree shaking

3. **Accessibility First**
   - Include ARIA labels and descriptions
   - Test with keyboard navigation
   - Verify screen reader compatibility
   - Maintain proper focus management

4. **Testing Excellence**
   - Write tests before implementing features (TDD)
   - Use descriptive test names and organize by behavior
   - Test user interactions, not implementation details
   - Maintain high test coverage with meaningful assertions

5. **Consistent Styling**
   - Use design tokens for all styling decisions
   - Follow established color and spacing patterns
   - Maintain consistent component APIs
   - Support both light and dark themes

The design system package provides a solid foundation for building consistent, accessible, and
maintainable user interfaces across the entire Forge platform with comprehensive testing standards
that ensure reliability and quality.
