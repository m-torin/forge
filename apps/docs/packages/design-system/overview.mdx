# Design System - Algolia Search

A focused collection of Algolia search components built with React InstantSearch
for powerful search functionality across applications.

## Overview

This package provides pre-built Algolia search components and utilities to
quickly implement instant search capabilities in your applications. Built on top
of React InstantSearch, it offers a set of composable components for building
search interfaces.

## Available Components

### Search Components

- **SearchProvider**: Context provider for Algolia InstantSearch
- **SearchBox**: Search input with query handling
- **SearchResults**: Display search results with customizable rendering
- **SearchStats**: Show search statistics (results count, query time)
- **Autocomplete**: Advanced autocomplete with suggestions

### React Hooks

- **useInstantSearch**: Access InstantSearch instance and methods
- **useInstantSearchConfig**: Configuration utilities for search setup
- **useSearchBox**: Search input state and query management
- **useSearchState**: Access current search state and parameters

### Utilities

- **createSearchClient**: Factory for creating configured Algolia search clients
- **getSearchConfig**: Helper for search configuration management

## Usage Examples

### Basic Search Interface

```typescript
import {
  SearchProvider,
  SearchBox,
  SearchResults,
  SearchStats
} from '@repo/design-system/algolia';

function ProductSearch() {
  const config = {
    applicationId: process.env.ALGOLIA_APPLICATION_ID!,
    searchApiKey: process.env.ALGOLIA_SEARCH_API_KEY!,
    indexName: 'products'
  };

  return (
    <SearchProvider config={config}>
      <div className="search-container">
        <SearchBox placeholder="Search products..." />
        <SearchStats />
        <SearchResults />
      </div>
    </SearchProvider>
  );
}
```

### Using Search Hooks

```typescript
import { useSearchBox, useInstantSearch } from '@repo/design-system/algolia';

function CustomSearchInput() {
  const { query, refine, clear } = useSearchBox();
  const { results } = useInstantSearch();

  return (
    <div>
      <input
        value={query}
        onChange={(e) => refine(e.target.value)}
        placeholder="Type to search..."
      />
      <button onClick={clear}>Clear</button>
      <p>{results?.nbHits} results found</p>
    </div>
  );
}
```

### Custom Search Client

```typescript
import { createSearchClient } from '@repo/design-system/algolia';

const customClient = createSearchClient({
  applicationId: 'YOUR_APP_ID',
  searchApiKey: 'YOUR_SEARCH_KEY',
  indexName: 'custom_index'
});

// Use with SearchProvider
<SearchProvider config={{ ...config, searchClient: customClient }}>
  {/* Your search components */}
</SearchProvider>
```

## Best Practices

1. **Configuration**: Store Algolia credentials in environment variables
2. **Performance**: Use appropriate `stalledSearchDelay` for your use case
3. **Routing**: Enable routing for shareable search URLs
4. **Customization**: Create custom hit components for your data structure
5. **Error Handling**: Implement error boundaries for search failures

## Limitations

- This package contains only Algolia search functionality
- For general UI components, use Mantine UI directly in your applications
- Custom design tokens and theming should be implemented at the application
  level
