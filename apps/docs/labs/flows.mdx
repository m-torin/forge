# Flows Lab

A visual programming platform for creating complex workflows and integrations. This lab explores business logic engines at scale using a drag-and-drop interface built with React Flow.

## Overview

The Flows lab is a complete workflow automation platform that enables users to create visual workflows with support for AWS services, AI providers, GitHub integration, and custom business logic.

- **React Flow Integration** - Advanced drag-and-drop interface

## Architecture

### Frontend
- **Next.js 15** - React framework with App Router
- **React Flow** - Visual workflow editor
- **TypeScript** - Type-safe development
- **SCSS Modules** - Component styling

### Backend
- **Prisma ORM** - Database schema and migrations
- **PostgreSQL** - Primary data storage
- **Redis** - Caching and session management
- **NextAuth.js** - Authentication with OAuth

### Security
- **Data Encryption** - Sensitive data protection
- **Rate Limiting** - API abuse prevention
- **WebAuthn Passkeys** - Passwordless authentication
- **GitHub OAuth** - Secure social login

## Environment Configuration

### Required Variables

```bash
# Database
PRISMA_DB_URL=postgresql://user:pass@localhost:5432/flows

# Authentication
AUTH_SECRET=your-auth-secret
GITHUB_ID=your-github-oauth-id
GITHUB_SECRET=your-github-oauth-secret

# AWS Integration
AWS_ACCESS_KEY_ID=your-aws-access-key
AWS_SECRET_ACCESS_KEY=your-aws-secret-key
AWS_REGION=us-east-1

# AI Services
OPENAI_API_KEY=sk-your-openai-key
ANTHROPIC_API_KEY=your-anthropic-key

# Cache & Storage
REDIS_URL=redis://localhost:6379
ENCRYPTION_SECRET=your-encryption-secret
```

### Optional Variables

```bash
# Vercel Deployment
VERCEL_AUTH_BEARER_TOKEN=your-vercel-token
VERCEL_PROJECT_ID=your-project-id

# Monitoring
SENTRY_DSN=your-sentry-dsn
```

## Quick Start

### 1. Setup Environment

```bash
# Clone and install
pnpm install

# Set up environment
cp env.example .env.local
# Edit .env.local with your values
```

### 2. Database Setup

```bash
# Generate Prisma client
pnpm prisma:generate

# Run migrations
pnpm prisma:migrate:dev

# (Optional) Seed data
pnpm prisma:seed
```

### 3. Start Development

```bash
# Start development server
pnpm dev

# Open http://localhost:3000
```

## Usage Examples

### Creating a Simple Workflow

1. **Add Trigger Node** - Start with a webhook or cron trigger
2. **Add Processing Node** - Use AI or logic nodes for data processing
3. **Add Output Node** - Send results via AWS SNS or store in S3
4. **Connect Nodes** - Link nodes with visual connections
5. **Test Workflow** - Use the built-in testing interface

### AWS Lambda Integration

```typescript
// Example Lambda node configuration
{
  "nodeType": "aws-lambda",
  "config": {
    "functionName": "processUserData",
    "payload": "{{ previousNode.output }}",
    "async": true
  }
}
```

### AI Processing Node

```typescript
// Example OpenAI node configuration
{
  "nodeType": "openai-gpt",
  "config": {
    "model": "gpt-4",
    "prompt": "Analyze the following data: {{ input.data }}",
    "temperature": 0.7,
    "maxTokens": 1000
  }
}
```

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Database Connection Issues">
    - Verify PostgreSQL is running
    - Check PRISMA_DB_URL format
    - Ensure database exists and is accessible
    - Run `pnpm prisma:migrate:deploy` for production
  </Accordion>
  
  <Accordion title="AWS Integration Problems">
    - Verify AWS credentials are correct
    - Check IAM permissions for required services
    - Ensure AWS region is properly configured
    - Test AWS connections in isolation
  </Accordion>
  
  <Accordion title="Flow Execution Failures">
    - Check node configurations for required fields
    - Verify API keys for external services
    - Review execution logs in the admin panel
    - Test individual nodes before full workflow
  </Accordion>
</AccordionGroup>
